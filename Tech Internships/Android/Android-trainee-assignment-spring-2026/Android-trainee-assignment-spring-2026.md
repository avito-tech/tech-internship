# **Тестовое задание для стажёра Android**

## **Описание задачи**
Нужно создать AI-ассистент на основе GigaChat для выполнения повседневных задач. Он должен работать в формате чата (переписки) и упрощать процессы и облегчать поиск информации.

## **Экран входа (логина)**

Экран авторизации пользователя через Firebase Authentication. Позволяет войти в систему с помощью E-mail и пароля (или через Google-аккаунт, опционально).

### **Состав экрана**

* **AppBar / верхняя панель**  
* Заголовок: «Вход в аккаунт»  
* Область ввода  
* Кнопка «Регистрация»

**Основная область входа**

* Поля ввода:  
  * E-mail  
  * Пароль

* Кнопка «Войти»

  * При нажатии запускает процесс авторизации через FirebaseAuth.

  * При успешном входе происходит навигация на главный экран с чатами.

  * При ошибке — показ Snackbar с текстом ошибки.

* Состояние загрузки:

  * При выполнении входа кнопка блокируется, вместо текста показывается `индикатор загрузки`.

* Поддержка автологина:

* При повторном запуске проверяется `FirebaseAuth.getCurrentUser()`.

* Если пользователь уже вошёл, сразу переход на главный экран.

### **Примечания**

* Валидация E-mail и пароля до отправки формы.

* Ошибки Firebase отображаются пользователю.

* В случае отсутствия сети сообщение об ошибке с кнопкой «Повторить».

* Поддержка светлой и тёмной темы.

* Возможность регистрации нового пользователя.

### **Дополнительно (опционально)**

* Добавить **анимацию появления** формы входа.   
* Добавить **вход через Google.**  
* Анимация логотипа приложения при запуске (Splash-экран).  
* Подсказки и переход фокуса между полями клавиатурой.  
* Состояние remember me или автовход.

## **Экран чата**

Для запросов к AI-ассистенту используем GigaChat:

- [Как получить токен](https://developers.sber.ru/studio/workspaces/019cdccb-1346-736a-a230-958b7131d985/gigachat-api/projects/019cdccb-5ea0-7cd1-9b70-a1a7c429f38c/settings)  
- [Запрос на генерация ответа](https://developers.sber.ru/docs/ru/gigachat/api/reference/rest/post-chat).

Экран предназначен для общения пользователя с AI-ассистентом в формате переписки.

**Экран состоит из следующих элементов:**

- Тулбар, который содержит:  
  - кнопку «Назад»  
  - заголовок чата  
- Список сообщений:  
  - сообщения пользователя  
  - сообщения ассистента.  
- Поле ввода сообщения, которое включает в себя:  
  - многострочное поле ввода  
  - кнопку отправки  
  - опционально кнопку очистки текста.

**Основной функционал**

- Отображать историю сообщений внутри выбранного чата.  
- Позволять пользователю вводить и отправлять сообщение.  
- Показывать состояние генерации ответа от нейросети.  
- Отображать ответ ассистента.  
- Поддерживать повторную отправку сообщения в случае ошибки.  
- Сохранять историю переписки между открытиями экрана.  
- Работать с длинными сообщениями и длинными списками сообщений.  
- Позволять поделиться ответом ассистента через системное меню Share.

**Дополнительно**

- Реализация генерации картинок.

**Что значит «поделиться ответом»**

Пользователь должен иметь возможность поделиться конкретным сообщением ассистента через стандартный Android Share Sheet.

Например, если нейросеть сгенерировала полезный ответ, пользователь нажимает кнопку «Поделиться», после чего открывается системное меню выбора приложения:

- Telegram  
- WhatsApp  
- Gmail  
- Заметки  
- и другие приложения, поддерживающие текстовый Share.

У каждого сообщения ассистента должна быть доступна кнопка «Поделиться», это может быть либо кнопка в интерфейсе, либо экшен на долгое нажатие сообщения, либо оба варианта.

## **Экран списка с чатами и темами**

**Экран состоит из следующих компонентов:**

- Floating Action Button для создания нового чата  
- Список уже созданных чатов  
- Строка поиска вверху экрана   
- Кнопка запуска поиска в правом верхнем углу экрана.

По нажатию на кнопку создания чата происходит навигация на экран с новым чатом.

Каждый элемент списка чатов содержит:

- Иконку чата (можно использовать любую подходящую png или векторную иконку)   
- Название чата.

Список чатов должен храниться локально на устройстве пользователя. Для хранения данных можно использовать файловое хранилище или базы данных, применяя библиотеку Room, SQLDeligt или встроенное SQLite API в Android SDK. Порядок отображения элементов можно выбрать самостоятельно (сортировка по названию чата, по времени создания).

Список чатов должен подгружаться порционно с  помощью пагинации. Когда пользователь прокручивает список до конца уже загруженных элементов, должна загружаться следующая порция данных. Размер порции данных можно выбрать самостоятельно.

По нажатию на кнопку поиска должен выполняться поиск по названиям чатов с применением введённого в строку текста. На экране должны отображаться только те чаты, названия которых соответствуют поисковому запросу. При нажатии на кнопку поиска с пустым запросом нужно отображать список без изменений, не запускать поиск. Поиск можно реализовать средствами выбранной библиотеки работы с файлами / базами данных (например, FTS3/FTS4), с использованием сторонней библиотеки или реализовав собственный поисковой механизм.

## **Экран профиля**

Предоставить пользователю возможность просматривать и редактировать базовую информацию своего аккаунта, привязанного к Firebase Authentication.

**Состав экрана:**

- AppBar  
- Заголовок «Профиль»  
- Фото пользователя (если нет, показываем заглушку)  
- Если фото нет → заглушка с иконкой пользователя  
- Информация о пользователе   
- Смена темы приложения  
- Кнопка выхода из аккаунта.

При клике на фото:

- Открыть системный выбор изображения (галерея/камера — опционально)  
- После выбора: загрузить изображение в приложение и в Firebase.

Информация о пользователе:

- Отображаем данные, доступные в FirebaseAuth:  
  - Имя пользователя  
  - E-mail  
  - Телефон, если учётная запись телефонная  
  - И количество токенов.

Режимы отображения:

- Минимум: поля только для просмотра  
- Опционально: возможность редактирования имени и фото.

Кнопки и действия:

- «Выйти из аккаунта» — разлогинить и перенаправить на экран Auth (обязательно реализовать).

Опционально:

- «Редактировать»/«Сохранить» (если нужно редактирование).
