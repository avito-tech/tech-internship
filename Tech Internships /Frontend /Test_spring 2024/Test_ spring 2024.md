### Описание

Тестовое необходимо реализовать с использованием [API Кинопоиска](https://api.kinopoisk.dev/documentation). Токен для доступа API  предоставлен отдельно в письме с почты interships@avito.ru.

Результат тестового должен соответствовать функциональным и нефункциональным требованиям описанным ниже. 

В пунктах "Будет плюсом" обозначены требования, выполнять которые не обязательно, но даже частичное выполнение этих пунктов позволит лучше раскрыть ваши знания и навыки.

### Функциональные требования

- Приложение должно состоять из нескольких страниц:
	- Страница со списком всех фильмов
	- Странийца отдельного фильма
###### Будет плюсом:
- Наличие авторизации (можно захардкодить проверку пароля), после авторизации открывается возможность найти рандомный фильм по фильтрам

#### Страница со списком всех фильмов

- Отображается список фильмов и сериалов
- Есть пагинация
- Можно выбрать сколько фильмов показывать на странице (по умолчанию - 10)
- Можно отфильтровать выдачу (по году, стране и возрастному рейтингу)
- Есть поиск по названию фильма
- Из выдачи можно перейти на страницу фильма
###### Будет плюсом:
- Можно поделиться своей выдаче через копирование ссылки. (записывать в query-параметры фильтры и пагинацию)
- История поиска сохраняется(последние 20 штук). При вводе нового названия появляется suggest с предложениями из ранее введенных значений. При вводе значений происходит фильтрация подсказок по вхождению. Например, при вводе бой: могут быть слова при**бой**, **бой**кий, беспере**бой**ность. (примерно как поиск у Google предлагает варианты предыдущих запросов)
- Поиск осуществляется не при каждом вводе символа, а когда с ввода последнего символа прошла 1s (debounce)

#### Страница фильма

- Отображается информация о фильме или сериале, в том числе:
	- Название
	- Постер(ы)
	- Описание
	- Рейтинг
	- Список актеров с пагиницией, если актеров больше 10
	- Список сезонов и серий (если есть) с пагинацией
	- Отзывы пользователей с пагинацией
	- Постеры отображаются в виде "карусели"
	- Выводится список фильмов, похожих на этот  в виде "карусели". По каждому элементу можно кликнуть и открыть его страницу
- В случае, если какой-то из списков пустой (отзывы, актеры, сезоны), нужно отображать заглушку на подобие "не информации о ..."
- Есть кнопка "назад", которая ведет на выдачу. Фильтры и номер страницы должны сохраниться.

###### Будет плюсом:
Страница c рандомным поиском([референс](https://www.kinopoisk.ru/chance/)):
- Есть возможность установить фильтры: тип, страна, сериал/фильм, поиск по году, поиск по рейтингу Кинопоиска(от какого-то значения), поиск по сети производства("HBO", "Netflix" и т.п.)
- Есть кнопка - "Случайный фильм", по клику на нее перекидывает на страницу найденого фильма


### Нефункциональные требования

- Стек технологий:
	- Фронтенд фреймворк - `react`, версия 18
	- Можно использовать любую общедоступную библиотеку компонент. Примеры: [ant.design](https://ant.design/), [mantine](https://mantine.dev/), [react-bootstrap](https://react-bootstrap.github.io/)
	- Сборщик - `Webpack`
	- `Node.js >= 18`, пакетный менеджер - `npm`
- Запуск проекта в режиме разработчика должен происходить по команде `TOKEN=<your api token> npm run start` , проект должен быть доступен по ссылке `http://localhost:7070`
- **Важно**, чтобы токена kinopoisk не было на `github` в вашем приложении
- С приложением должно быть удобно работать, как с мобильного экрана, так и с десктопа (адаптивный интерфейс)
- Роутинг выполнен с использованием [React Router v6](https://reactrouter.com/en/main)
- При переходах по ссылкам страница не перезагружается (SPA, без next.js)
###### Будет плюсом:
- Использование `TypeScript`
- Docker-файл для запуска
- При неудачном запросе должно быть три попытки повторного запроса
- При переходе со страницы на страницу запросы, относящиеся к старой странице, должны прерываться (отменяться/прекращаться)
- Покрытие кода юнит-тестами
